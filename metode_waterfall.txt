Pengertian:  Metode Waterfall merupakan pendekatan SDLC paling awal yang digunakan untuk pengembangan perangkat lunak.
Ian Sommerville (2011) menjelaskan bahwa ada lima tahapan pada Metode Waterfall, yakni Requirements Analysis and Definition, Sytem and Software Design, Implementation and Unit Testing, Integration and System Testing, dan Operationa and Maintenance.
Tahapan:
- Requirement Analysis: mempersiapkan dan menganalisa kebutuhan dari software yang akan dikerjakan
- System and Software Design:  pembuatan desain aplikasi sebelum masuk pada proses coding
- Implementation and Unit Testing: implementasi kode program dengan menggunakan berbagai tools dan bahasa pemrograman sesuai dengan kebutuhan tim dan perusahaan
- Integration and System Testing: masuk dalam proses integrasi dan pengujian sistem
- Operation and Maintenance: pengoperasian dan perbaikan dari aplikasi
